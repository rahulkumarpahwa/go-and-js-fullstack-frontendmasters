What is the purpose of using pushState in the History API?
PushState allows changing the URL in the browser's address bar without reloading the page, creates a new history entry, and enables client-side routing while maintaining browser navigation capabilities like back and forward buttons

What does location.pathname provide?
Location.pathname returns the current URL path without the domain, protocol, or query parameters

What is the purpose of the popState event?
The popState event is fired when the user navigates through browser history, such as clicking the back or forward buttons, allowing the application to respond to navigation changes

Why might you not want to add a new history entry when navigating?
In certain scenarios like login forms, adding a new history entry might not make sense. For example, after logging in, you don't want users to be able to go back to the login page by pressing the back button

What does location.search represent?
Location.search returns the query string portion of the URL, which includes everything after the '?' character, such as parameters and their values
